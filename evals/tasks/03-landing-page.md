# Task: Landing Page for "ChronoLog"

## The Prompt

Create a landing page for "ChronoLog" — a developer tool that automatically generates changelogs from git history using AI. It analyzes commits, groups them by theme, and produces human-readable release notes.

The page should:
- Explain what ChronoLog does and why someone would want it
- Show how it works (not just what it does)
- Have a clear call to action
- Be a single HTML file with inline CSS (no frameworks, no build step)

That's it. Make your own decisions on structure, copy, and design.

## Why This Task

This task has genuine creative slots:
- **Narrative angle**: Developer productivity vs release management vs team communication
- **Structure**: Hero + features + CTA vs problem/solution vs story-driven
- **Tone**: Technical/precise vs casual/friendly vs bold/opinionated
- **Social proof approach**: Code examples vs testimonials vs before/after comparisons
- **Visual style**: Minimal/typographic vs colorful/illustrated vs dark-mode developer aesthetic

These choices reflect different theories about what persuades developers to try a tool.

## Acceptance Criteria

The output must be:
1. A single index.html file that opens in a browser and looks good
2. Copy that accurately describes what ChronoLog does
3. A visitor can understand the product and know what to do next within 30 seconds

## Evaluation Dimensions

| Dimension | What to look for |
|-----------|-----------------|
| **Clarity** | Can a visitor understand the product in 10 seconds? |
| **Persuasiveness** | Would a developer actually click the CTA? |
| **Approach creativity** | Did the process surface non-obvious angles or framings? |
| **Visual design** | Does it look professional? Consistent? Not template-y? |
| **Copy quality** | Concise, specific, no fluff, no AI-slop? |
| **Technical accuracy** | Does the page itself demonstrate good HTML/CSS craft? |
